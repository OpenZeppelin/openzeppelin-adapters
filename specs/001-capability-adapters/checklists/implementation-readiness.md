## Scenario Coverage ‚Äî Edge Cases

- [x] CHK040 ‚Äî Are requirements for concurrent `dispose()` calls defined ‚Äî is it idempotent, or does double-dispose throw? [Edge Case, Spec ¬ßFR-010] ‚Äî **Addressed**: Spec edge cases and data model both state "`dispose()` is idempotent ‚Äî calling it twice is a no-op (does not throw)."
- [x] CHK041 ‚Äî Are requirements defined for adapter packages that implement a capability but only partially (e.g., `ExplorerCapability` with `getExplorerUrl` but not `getExplorerTxUrl` which is optional)? An adapter satisfies the interface by implementing all required methods; optional methods may be omitted. This is standard TypeScript interface compliance.
- [x] CHK042 ‚Äî Are requirements for error messages specified when a consumer accesses a capability that exists on the runtime type but was not included in the requested profile (TypeScript type vs runtime availability)? [Edge Case, Gap] ‚Äî **Addressed**: Added to spec edge cases ‚Äî "The property is `undefined` at runtime. TypeScript's type system reflects this via optional properties on profile-specific runtime types."
